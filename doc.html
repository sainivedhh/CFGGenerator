01 Lexical Analysis

The lexer scans raw source code sequentially and converts streams of characters into primitive Tokens. The studio specifically recognizes identifiers, core datatypes (int, float, char), and standard control-flow keywords.

  • Keywords: Control structures and types.
  • Identifiers: Variables and function names.
  • Literals: Numbers and strings.

02 Abstract Syntax Tree (AST)

The recursive-descent parser consumes the tokenised stream and generates a hierarchical Abstract Syntax Tree validating syntactic correctness. The UI renders this structure using dynamic SVG cubic-bézier curves.

Nested scopes and block-level operations are safely grouped under Block nodes.

03 Semantic Analysis

Post-parsing, the semantic analyser traverses the AST in post-order. It establishes a robust Symbol Table mapping variables to their scope and types, actively guarding against undeclared variables or datatype mismatched initializations.

  • Type Safety: Prevents assigning a char* to an int.
  • Scope Integrity: Prevents usage of undeclared identifiers.

04 Intermediate Representation (IR)

The compiler lowers the complex hierarchical AST down to a linear language—Three-Address Code (TAC). Expressions are flattened into temporary variables (t1, t2) and control flow transforms into explicit jumps (goto) and labels (L1:).

05 Control-Flow Graph (CFG)

The TAC is split into pure Basic Blocks (sequences of instructions with zero branching or halt parameters except at the very exit). From these blocks, directed edges calculate execution control flow—visualizing logical paths explicitly.

This includes detection for loops ("Back-Edges") and conditional branches ("True/False edges").
